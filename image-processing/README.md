# Image Processing for Object Detection and Feature Extraction

## Overview
This project focused on fundamental image processing techniques for object detection and shape analysis in MATLAB. The work was divided into two main tasks:

- detecting selected fruits in a colour image and computing their **bounding boxes** and **centroids**
- extracting geometric features from a separate object, including its **centroid**, **orientation**, and **circularity**

The project combined threshold-based segmentation, binary image analysis, moment calculations, inertia-matrix-based orientation estimation, and edge detection. A key motivation throughout the work was the use of image processing in robotics, particularly for applications such as object localisation and robotic grasping. :contentReference[oaicite:5]{index=5} :contentReference[oaicite:6]{index=6}

## Task 1: Bounding Boxes and Centroids of Fruits

### Objective
The first task was to isolate and analyse three objects in a fruit-bowl image:
- apple
- lemon
- capsicum. :contentReference[oaicite:10]{index=10}

The objective was to identify each fruit separately, create a binary image for it, then compute:
- a **bounding box**
- and its **centroid**. :contentReference[oaicite:11]{index=11}

### Colour Thresholding and Segmentation
The report explains that each fruit was isolated by exploiting its colour differences in the **red, green, and blue channels**. Separate thresholds were selected for each fruit so that pixels matching the target colour were set to white and all others to black, producing a binary image for further analysis. :contentReference[oaicite:12]{index=12} :contentReference[oaicite:13]{index=13}

This is a simple but important computer vision idea: rather than trying to recognise the whole object directly, the image is first reduced to a cleaner binary representation where object detection becomes much easier.

### Bounding Box Detection
The bounding box was generated by locating the:
- topmost
- bottommost
- leftmost
- and rightmost

white pixels in the thresholded image, then constructing a rectangle that enclosed the full detected region. The report correctly explains that this provides the first stage of object recognition by localising the target object spatially within the image. :contentReference[oaicite:14]{index=14}

### Centroid Estimation
Two centroid methods were used:

- **midpoint of the bounding box**
- **moment method**. :contentReference[oaicite:15]{index=15}

The report explains that the moment method is based on image moments analogous to centre-of-gravity calculations in mechanics. This is a stronger method for irregular or partially segmented objects because it reflects the spatial distribution of white pixels rather than assuming the object is centred in its bounding box. :contentReference[oaicite:16]{index=16}

A useful conclusion from the results is that:
- for the **lemon**, the two centroid methods were close because the binary shape was more evenly distributed
- for the **apple** and **capsicum**, the centroids differed more because of uneven pixel distribution, colour variation, stems, reflections, and non-uniform segmentation. :contentReference[oaicite:17]{index=17} :contentReference[oaicite:18]{index=18}

That is a strong point to include in a portfolio because it shows you were not just applying formulas blindly — you were evaluating why different methods behave differently on real images.

## Task 2: Blob and Feature Extraction

### Objective
The second task moved beyond localisation and into **feature extraction**. For a separate object image, the required outputs were:
- centroid
- orientation
- circularity. :contentReference[oaicite:19]{index=19}

This makes the task more relevant to robotics and automation, because identifying where an object is is not always enough — many systems also need to know **how it is oriented** and **what kind of shape it has**.

### Binary Object Representation
As in Task 1, the first step was to create a black-and-white image of the object using thresholding. The report notes that the object was black, but also contained bright reflections, so thresholds had to be chosen carefully to isolate the object while reducing the effect of unwanted highlights. :contentReference[oaicite:20]{index=20}

### Centroid of the Object
For this task, the **moment method** was used directly, since it is more reliable for irregular shapes than simply using a box midpoint. The report explicitly links this to robotic-arm applications, where a good estimate of the object centre is important for successful grasping. :contentReference[oaicite:21]{index=21}

### Orientation via Inertia Matrix
The report then derives the object orientation using:
- **central moments**
- the **inertia matrix**
- and the **eigenvalues/eigenvectors** of that matrix. :contentReference[oaicite:22]{index=22}

This is one of the strongest parts of the project technically. Instead of using a simple visual estimate, the orientation is derived analytically from the spatial distribution of the binary object pixels. The report also explains that the object is approximated by an equivalent ellipse, whose major axis gives the object orientation. :contentReference[oaicite:23]{index=23}

That is exactly the kind of mathematical-to-practical link that looks strong in a portfolio.

### Circularity Estimation
To quantify shape, the report calculates **circularity** using area and perimeter. The area comes from the binary object, while the perimeter is estimated using **edge detection**. In the strongest report version, this is explained using a **Sobel gradient filter**, where brightness changes are used to extract the boundary of the object before summing edge pixels to estimate perimeter. :contentReference[oaicite:24]{index=24}

The report then compares the measured circularity with a theoretical approximation based on a rectangle-like interpretation of the object, and concludes that the result is reasonable for the observed shape. :contentReference[oaicite:25]{index=25}

This is another strong feature of the project: you did not just compute a metric, but checked whether it was physically meaningful.
